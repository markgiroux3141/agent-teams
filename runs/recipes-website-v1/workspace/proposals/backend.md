# Backend Proposal

## My slice

I'm building:
1. **Recipe data schema** — a JSON structure that covers writer's content needs (name, category, time, servings, description, ingredients, steps, optional difficulty/dietary_tags)
2. **Data-loading mechanism** — frontend will embed a `<script type="application/json" id="recipes-data">` block in index.html containing the full recipes array. app.js reads it via `document.getElementById('recipes-data').textContent` and parses on page load. This works from file:// with no server.
3. **Search + category filter logic** — app.js wires the search input (`#search-input`) and category filter buttons (`.rx-filter-button[data-category]`) to re-render the recipe list. Filters apply together (AND logic: category AND search term both match).
4. **Modal detail view logic** — clicking a recipe card (`.rx-card`) opens the detail modal (`#recipe-detail`), populated with full ingredients and steps. Clicking outside the modal closes it.

## Schema

```json
{
  "id": "string (required, unique identifier)",
  "name": "string (required)",
  "description": "string (required, 1–2 sentences)",
  "category": "string (required, one of: breakfast, main, dessert)",
  "time_minutes": "number (required, integer)",
  "servings": "number (required, integer)",
  "ingredients": "array of strings (required, e.g. '2 cups flour')",
  "steps": "array of strings (required, ordered instructions)",
  "difficulty": "string (optional, one of: easy, medium, hard)",
  "dietary_tags": "array of strings (optional, e.g. 'vegetarian', 'gluten_free')",
  "image_emoji": "string (optional, single unicode character for category marker)"
}
```

Example recipe:
```json
{
  "id": "scrambled-eggs",
  "name": "Scrambled Eggs",
  "description": "Soft, creamy eggs. Takes 5 minutes. Perfect for breakfast.",
  "category": "breakfast",
  "time_minutes": 5,
  "servings": 2,
  "ingredients": ["3 large eggs", "1 tablespoon butter", "salt and pepper to taste"],
  "steps": ["Crack eggs into a bowl and whisk.", "Melt butter in a pan over medium heat.", "Pour in eggs and stir constantly until cooked through.", "Season and serve."],
  "difficulty": "easy",
  "dietary_tags": ["vegetarian"],
  "image_emoji": "🥚"
}
```

## Implementation details

**app.js will:**
- Load recipes on page load: `const data = JSON.parse(document.getElementById('recipes-data').textContent); const recipes = data;`
- Maintain a "filtered" state (current search term, selected categories).
- On search input, filter recipes by name/description/ingredients and re-render.
- On filter button click, toggle the category (multi-select) and re-render.
- On recipe card click, show detail modal with full recipe.
- On modal background click, hide modal.
- No external libraries. Vanilla DOM manipulation.

**Constraints addressed:**
- Works from `file://` — no fetch, data is embedded in HTML.
- No build step — frontend manually embeds JSON into script tag, I read it raw.
- No external JS libraries.
